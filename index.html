JENIS-JENIS CONSTRUCTOR

Dart menyediakan beberapa mekanisme inisialisasi objek untuk berbagai kebutuhan arsitektur:

  1. Default Constructor: ClassName()
  2. Named Constructor: ClassName.name()
  3. Parameterized Constructor: ClassName(this.param)
  4. Constant Constructor: const ClassName()
  5. Factory Constructor: (Akan dibahas pada sesi lanjut)

2. NAMED CONSTRUCTOR - CONTOH DETAIL

Dart memungkinkan pembuatan lebih dari satu konstruktor dalam satu kelas menggunakan Named Constructor. Ini sangat berguna untuk memberikan cara alternatif dalam menginisialisasi objek, seperti dari data JSON. 

// Nama   : Azriel Miracle Yuandre
// Nim    : 251410024
// Kelas  : Si2A

import 'dart:convert';

class Mobil {
  String? merk;
  int? tahun;

  // Parameterized constructor
  Mobil(this.merk, this.tahun);

  // Named constructor
  Mobil.baru() {
    merk = 'Bmw';
    tahun = 2023;
  }

  // Named constructor dengan parameter
  Mobil.dariJson(Map<String, dynamic> json) {
    merk = json['merk'];
    tahun = json['tahun'];
  }

  // Override toString for better print output
  @override
  String toString() {
    return 'Mobil{merk: $merk, tahun: $tahun}';
  }
}

void main() {
  print('--- Demonstrasi Class Mobil ---');

  // 1. Menggunakan constructor default (parameterized constructor)
  var mobilA = Mobil('Mercedes', 2020);
  print('Mobil A (constructor default): $mobilA');

  // 2. Menggunakan named constructor Mobil.baru()
  var mobilB = Mobil.baru();
  print('Mobil B (named constructor .baru): $mobilB');

  // 3. Menggunakan named constructor Mobil.dariJson()
  String jsonString = '{"merk": "Jaguar", "tahun": 2021}';
  Map<String, dynamic> jsonData = jsonDecode(jsonString);
  var mobilC = Mobil.dariJson(jsonData);
  print('Mobil C (named constructor .dariJson): $mobilC');

  // Contoh lain dengan named constructor Mobil.dariJson()
  Map<String, dynamic> anotherJsonData = {'merk': 'Audi', 'tahun': 2022};
  var mobilD = Mobil.dariJson(anotherJsonData);
  print('Mobil D (named constructor .dariJson): $mobilD');

  print('\n--- Akhir Demonstrasi ---');
}

3. Constant Constructor

Konstruktor konstan digunakan untuk membuat objek yang immutable (tidak bisa diubah). Objek yang dibuat dengan const dan nilai yang sama akan menggunakan instance yang sama di memori, sehingga lebih efisien. 

// Nama   : Azriel Miracle Yuandre
// Nim    : 251410024
// Kelas  : Si2A

class Warna {
  final int r, g, b;
  const Warna(this.r, this.g, this.b);

  @override
  String toString() {
    return 'Warna(r: $r, g: $g, b: $b)';
  }
}

void main() {
  const merah = Warna(255, 0, 0);
  const merahLagi = Warna(255, 0, 0); // Menggunakan instance yang sama di memory

  print(merah);
  print(merahLagi);
  print('Apakah instance sama? ${identical(merah, merahLagi)}');
}

PENGANTAR ENKAPSULASI

Konsep Enkapsulasi

Konsep: Menyembunyikan detail implementasi dari luar.

Tujuan:

  • Keamanan data
  • Kontrol akses
  • Fleksibilitas perubahan

Cara di Dart: Menggunakan underscore (_) untuk private members.

Public: Dapat diakses dari mana saja (default).

Private: Hanya dapat diakses dalam file yang sama.

// Nama   : Azriel Miracle Yuandre
// Nim    : 251410024
// Kelas  : Si2A 

class Person {
  String _name; // Private: Hanya bisa diakses di file ini
  int age;      // Public: Dapat diakses bebas

  Person(this._name, this.age);

  void info() => print('Nama: $_name, Umur: $age');
}

void main() {
  var p = Person('Azriel Miracle Yuandre', 20);
  p.info();
}

Getter And Setter

Pengertian Getter dan Setter

Getter: Memungkinkan membaca data private dengan kontrol.

Sintaks 1: tipeData get namaGetter => _field; (Arrow)

Sintaks 2: tipeData namaGetter() { return _field; } (Block)

Setter: Memungkinkan modifikasi data dengan validasi.

Sintaks 1: set namaSetter(value) {...} (Set Keyword)

Sintaks 2: void namaSetter(value) {...} (Method Approach)

// Nama   : Azriel Miracle Yuandre
// Nim    : 251410024
// Kelas  : Si2A 

class Person {
  String _name;
  int age;
  
  Person(this._name, this.age);
  
  set setName1(String name) {
    _name = name;
  }

  void setName(String name) {
    _name = name;
  }

  String getName() {
    return _name;
  }
}

void main() {
  Person person = Person("Azriel Miracle Yuandre", 20);
  print('Umur: ${person.age}');
  
  person.setName('Miracle');
  print('Nama (via Method): ${person.getName()}');
  
  person.setName1 = 'Azriel Miracle Yuandre';
  print('Nama (via Setter): ${person.getName()}');
}

class Produk {
  String _nama;
  double _harga;
  
  Produk(this._nama, this._harga);
  
  String get nama => _nama;
  double get harga => _harga;
  
  set nama(String value) {
    if (value.length >= 3) {
      _nama = value;

      print('Gagal memperbarui nama: Nama harus memiliki minimal 3 karakter.');

  set harga(double value) {
    if (value > 0) {
      _harga = value;
    } else {
      print('Gagal memperbarui harga: Harga harus lebih besar dari 0.');
    }
  }
}

class User {
  String _username;
  String _password;
  String _email;

  User(this._username, this._password, this._email);

  // Getter untuk semua field
  String get username => _username;
  String get email => _email;

  // Setter dengan validasi
  set username(String v) {
    if (v.length >= 5) {
      _username = v;
    } else {
      print('Username minimal 5 karakter');
    }
  }

  set password(String v) {
    if (v.length >= 8) {
      _password = v;
    } else {
      print('Password minimal 8 karakter');
    }
  }

  set email(String v) {
    if (v.contains('@')) {
      _email = v;
    } else {
      print('Format email salah (Harus mengandung @)');
    }
  }

  void info() => print('User: $_username, Pass: $_password, Email: $_email');
}

void main() {
  var user = User('Azriel', '10001', 'admin.com');
  user.username = 'Azrielmy';
  user.password = 'thegreat1';
  user.email = 'Azriel@gmail.com';
  user.info();
}

class User {
  String _username;
  String _password;

  User(this._username, this._password);
  
  String get username => _username;

  bool login(String username, String password) {
    return _username == username && _password == password;
  }

  void gantiPassword(String lama, String baru) {
    if (_password == lama) {
      if (baru.length >= 8) {
        _password = baru;
        print('Password Anda berhasil diubah.');
      } else {
        print('Gagal: Password baru minimal 8 karakter.');
      }
    } else {
      print('Gagal: Password lama tidak cocok.');
    }
  }
}

void main() {
  print('--- Inisialisasi User ---');
  final User user1 = User('admin_Azriel', 'securePass123');
  
  print('--- Uji Coba Login ---');
  if (user1.login('admin_Azriel', 'securePass123')) {
    print('Login Anda Berhasil!');
  }

  print('\n--- Uji Coba Ganti Password ---');
  user1.gantiPassword('securePass123', 'newSecurePass456');
  user1.gantiPassword('wrongPass', 'anyPass'); // Gagal
}

class RekeningBank {
  String _nomorRekening;
  double _saldo;

  RekeningBank(this._nomorRekening, this._saldo);

  void deposit(double jml) {
    if (jml > 0) {
      _saldo += jml;
      print('Deposit Rp $jml Berhasil.');
    }
  }

  void tarik(double jml) {
    if (jml > 0 && _saldo >= jml) {
      _saldo -= jml;
      print('Tarik Rp $jml Berhasil.');
    } else {
      print('Gagal: Saldo tidak cukup atau input tidak valid.');
    }
  }

  void cekSaldo() {
    print('Rekening: $_nomorRekening | Saldo: Rp $_saldo');
  }
}

void main() {
  var bank = RekeningBank('9837-10009', 23000000);
  bank.cekSaldo();
  bank.tarik(500000);
  bank.cekSaldo();
}

Kesimpulan

Berdasarkan pembahasan dan implementasi yang telah dilakukan, dapat disimpulkan bahwa Object Oriented Programming (OOP) merupakan paradigma pemrograman yang efektif dalam membangun sistem perangkat lunak yang terstruktur dan modular. Dengan menggunakan konsep class dan object, program dapat direpresentasikan secara lebih sistematis sesuai dengan model dunia nyata.

Penerapan constructor memastikan bahwa setiap objek yang dibuat memiliki kondisi awal yang valid, sehingga mengurangi potensi kesalahan selama proses eksekusi. Sementara itu, validasi pada constructor berperan penting dalam menjaga konsistensi data dan mencegah terjadinya nilai yang tidak logis dalam sistem.

Melalui prinsip encapsulation, data internal objek dapat dilindungi dari akses langsung yang tidak sah, sehingga meningkatkan keamanan dan stabilitas program. Penggunaan named parameter dengan keyword required juga membantu meningkatkan kejelasan kode dan meminimalkan kesalahan input saat proses instansiasi objek.

Secara keseluruhan, penerapan OOP dalam bahasa Dart menunjukkan bahwa pendekatan ini mampu meningkatkan keterbacaan kode, efisiensi pengembangan, serta kemudahan pemeliharaan sistem dalam jangka panjang. Paradigma ini menjadi fondasi penting dalam pengembangan aplikasi modern, termasuk dalam framework Flutter yang berbasis Dart.

Dengan memahami dan mengimplementasikan konsep-konsep dasar OOP seperti class, object, constructor, validasi, dan encapsulation, pengembang dapat membangun sistem yang lebih robust, scalable, dan profesional.
