2. NAMED CONSTRUCTOR-CONTOH DETAIL

Dart memungkinkan pembuatan lebih dari satu konstruktor dalam satu kelas menggunakan Namest Constructor. Ini sangat berguna untuk memberikan cara alternatif dalam menginisialisasi objek, seperti dari data JSON. 

// Nama   : Azriel Miracle Yuandre
// Nim    : 251410024
// Kelas  : Si2A

import 'dart:convert';

class Mobil {
  String? merk;
  int? tahun;

  // Parameterized constructor
  Mobil(this.merk, this.tahun);

  // Named constructor
  Mobil.baru() {
    merk = 'Bmw';
    tahun = 2023;
  }

  // Named constructor dengan parameter
  Mobil.dariJson(Map json) {
    merk = json['merk'];
    tahun = json['tahun'];
  }

  // Override toString for better print output
  @override
  String toString() {
    return 'Mobil{merk: $merk, tahun: $tahun}';
  }
}

void main() {
  print('--- Demonstrasi Class Mobil ---');

  // 1. Menggunakan constructor default (parameterized constructor)
  var mobilA = Mobil('Mercedes', 2020);
  print('Mobil A (constructor default): $mobilA');

  // 2. Menggunakan named constructor Mobil.baru()
  var mobilB = Mobil.baru();
  print('Mobil B (named constructor .baru): $mobilB');

  // 3. Menggunakan named constructor Mobil.dariJson()
  String jsonString = '{"merk": "Jaguar", "tahun": 2021}';
  Map jsonData = jsonDecode(jsonString);
  var mobilC = Mobil.dariJson(jsonData);
  print('Mobil C (named constructor .dariJson): $mobilC');

  // Contoh lain dengan named constructor Mobil.dariJson()
  Map anotherJsonData = {'merk': 'Audi', 'tahun': 2022};
  var mobilD = Mobil.dariJson(anotherJsonData);
  print('Mobil D (named constructor .dariJson): $mobilD');

  print('\n--- Akhir Demonstrasi ---');
}

3. Validasi Constructor (Mahasiswa)

CONSTANT CONSTRUCTOR Konstruktor konstan digunakan untuk membuat objek yang immutable (tidak bisa diubah). Objek yang dibuat dengan const dan nilai yang sama akan menggunakan instance yang sama di memori, sehingga lebih efisien. 

// Nama   : Azriel Miracle Yuandre
// Nim    : 251410024
// Kelas  : Si2A

class Warna {
  final int r, g, b;
  const Warna(this.r, this.g, this.b);

  @override
  String toString() {
    return 'Warna(r: $r, g: $g, b: $b)';
  }
}

void main() {
  const scarlet = Warna(255, 0, 0);
  const scarlet = Warna(255, 0, 0); // Menggunakan instance yang sama di memory

  print(scarlet);
  print(scarlet);
}

4. CONSTRUCTOR DENGAN INITIALIZER LIST

Initializer List digunakan untuk menginisialisasi field sebelum body konstruktor dijalankan. Ini sering digunakan untuk melakukan validasi atau transformasi data pada field final. 

// Nama   : Azriel Miracle Yuandre
// Nim    : 251410024
// Kelas  : Si2A

class Titik {
  final double x, y;

  Titik(double x, double y)
    : x = x.abs(),
      y = y.abs() {
    print('Titik dibuat di (${this.x}, ${this.y})'); // Using this.x and this.y to show the absolute values
    
  }
}

void main(){
  // Contoh penggunaan class Titik
  print('Membuat objek Titik pertama:');
  var titik1 = Titik(3.0, 4.0); // x=3.0, y=4.0
  print('Titik 1: x = ${titik1.x}, y = ${titik1.y}');

  print('\nMembuat objek Titik kedua dengan nilai negatif:');
  var titik2 = Titik(-5.0, 10.0); // x=5.0 (abs), y=10.0 (abs)
  print('Titik 2: x = ${titik2.x}, y = ${titik2.y}');

  print('\nMembuat objek Titik ketiga dengan nilai negatif di kedua koordinat:');
  var titik3 = Titik(-1.5, -2.5); // x=1.5 (abs), y=2.5 (abs)
  print('Titik 3: x = ${titik3.x}, y = ${titik3.y}');
}

5. PRAKTIK 1-MULTIPLE CONSTRUCTOR

Latihan pembuatan kelas Mahasiswa dengan tiga jenis konstruktor berbeda untuk fleksibilitas instansiasi objek sesuai kebutuhan data: 

// Nama   : Azriel Miracle Yuandre
// Nim    : 251410024
// Kelas  : Si2A

class Mahasiswa {
  String? nama;
  String? nim;

  Mahasiswa(this.nama, this.nim);

  Mahasiswa.dariMap(Map data) {
    nama = data['nama'];
    nim = data['nim'];
  }

  Mahasiswa.tanpaNama() {
    nama = 'Nor Name';
    nim = '000001';
  }

  void info() => print('Mahasiswa: $nama, NIM: $nim');
}

void main() {
  var m1 = Mahasiswa('Azriel', '251410024');
  var m2 = Mahasiswa.dariMap({'nama': 'Miracle', 'nim': '251410024'});
  var m3 = Mahasiswa.tanpaNama();

  m1.info();
  m2.info();
  m3.info();
}

Sementara itu, validasi pada constructor berperan penting dalam menjaga konsistensi data dan mencegah terjadinya nilai yang tidak logis dalam sistem.

Melalui prinsip encapsulation, data internal objek dapat dilindungi dari akses langsung yang tidak sah, sehingga meningkatkan keamanan dan stabilitas program. Penggunaan named parameter dengan keyword required juga membantu meningkatkan kejelasan kode dan meminimalkan kesalahan input saat proses instansiasi objek.

Secara keseluruhan, penerapan OOP dalam bahasa Dart menunjukkan bahwa pendekatan ini mampu meningkatkan keterbacaan kode, efisiensi pengembangan, serta kemudahan pemeliharaan sistem dalam jangka panjang. Paradigma ini menjadi fondasi penting dalam pengembangan aplikasi modern, termasuk dalam framework Flutter yang berbasis Dart.

Dengan memahami dan mengimplementasikan konsep-konsep dasar OOP seperti class, object, constructor, validasi, dan encapsulation, pengembang dapat membangun sistem yang lebih robust, scalable, dan profesional.
